# Navicat NCX 连接导入设计

## 目标

在现有连接管理中增加“导入 Navicat 连接”入口。用户选择 Navicat 导出的 `.ncx` 文件后，先预览可识别连接，再一键批量导入；不改变现有连接结构、路由或数据库连接逻辑。

## 数据流

1. 渲染进程通过专用原生文件选择器选取 `.ncx`。
2. 主进程读取并解析 XML，将规范化结果保存在与当前窗口绑定、十分钟有效的临时会话中。
3. 渲染进程只获得脱敏预览，不接触密码或可篡改的最终配置。
4. 用户勾选连接并确认后，主进程按会话中的数据执行一次原子批量写入。
5. 完全重复的连接跳过；仅名称冲突的连接自动追加“(Navicat)”序号。

## 兼容与安全

- 支持 MySQL/MariaDB、PostgreSQL、SQL Server、SQLite、ClickHouse、OceanBase 与 OceanBase Oracle。
- XML 使用严格解析，限制为 20 MB、20,000 个节点、64 层、1,000 个连接，并拒绝 DTD/ENTITY。
- 兼容 UTF-8、UTF-16LE 和 UTF-16BE BOM。
- 自动兼容 NCX 的 V2 AES-128-CBC 与 V1 Blowfish 密码格式，数据库密码、SSH 密码和私钥口令均在主进程解密。
- 解密后的明文不返回渲染进程；批量写入时立即转为 Electron `safeStorage` 密文。非空密码若无法解密，对应连接明确跳过，不以空密码导入。
- 若配置包含 SQLite 文件或 SSH 私钥路径，写入前显示原生可信确认框。
- 不支持的数据库只在预览中提示并跳过，不中断其他连接。

## 界面

文件菜单新增“导入 Navicat 连接…”。预览弹窗显示名称、类型、目标、分组和状态；可识别且非重复项默认勾选。成功后刷新连接树并显示导入、跳过、重命名数量。

## 验证

自测覆盖 NCX 类型映射、分组、SSH、SQLite、密码提示、重复跳过、名称冲突、危险 XML 和批量原子写入；随后执行完整 `npm run verify` 与目录打包。
