大模型API使用说明

获取API密钥

1. 登录 [万象MaaS平台](https://llm.thunisoft.com/#/auth/login)
2. 点击侧边栏 应用接入 → [应用管理 ](https://llm.thunisoft.com/#/maas/project/app/application)，在页面中复制 API Key

API接口

openai 兼容接口地址：

Base URL`https://llm.thunisoft.com/v1`复制

Chat`https://llm.thunisoft.com/v1/chat/completions`复制

使用提示

- 请妥善保管您的API密钥
- 在生产环境中建议开启超时、重试和请求日志，便于排查问题
- 建议查看模型广场可用模型列表，确认模型是否支持多模态或函数调用
- `/v1/models` 与 `/v1/usage` 都使用应用 API Key 鉴权（Bearer）
- 流式返回时请优先处理 delta.content 与 delta.reasoning_content 字段

用户端模型 Token 限流

平台在**用户工作空间维度**会对调用进行模型级 Token 限流， 生效配额可来自工作空间专属或模型默认，未命中任何生效配额的模型在界面上会归入「无限流」类。

在 [应用管理 ](https://llm.thunisoft.com/#/maas/project/app/application)中点击 「模型 Token 限流」 ， 可打开前用户工作空间限流看板，按模型查看限额与已用/剩余，「有限流」与「无限流」分类。

- 当任一时间窗内用量超过生效上限时，接口会返回 `429 Too Many Requests`（请求过多，已被限流），请以实际返回为准。
- 出现 429 时建议：降低并发、避免短时突发；待对应时间窗滚动后再试，或对重试做指数退避/抖动，不要对 429 进行无间隔重试，以免持续触发限流，可结合弹窗中各模型的「剩余」额度规划调用。

调用步骤（推荐）

1. 获取 API Key
2. 选定模型后调用 `/chat/completions` 完成对话
3. 如需实时输出，设置 `stream=true`；如需工具能力，传入 `tools`
4. 基础对话示例（普通请求）

cURLJavaScriptPython复制代码

```
from openai import OpenAI

# 替换为您的应用代码
app_code = "您的应用代码"

# 创建客户端实例
client = OpenAI(
    base_url="https://llm.thunisoft.com/v1",
    api_key=app_code,  # 将应用代码作为 API Key
    max_retries=1
)

# 普通请求示例
response = client.chat.completions.create(
    model="my-model",  # 指定模型名称，必填项
    messages=[
        {"role": "system", "content": "你是一个有帮助的助手"},
        {"role": "user", "content": "你好，请介绍一下自己。"}
    ]
)

# 输出结果
if response.choices[0].message.reasoning_content:
    print("reasoning_content:")
    print(response.choices[0].message.reasoning_content)
print("content:")
print(response.choices[0].message.content)
```

2) 流式输出示例（stream=true）

cURLJavaScriptPython复制代码

```
# 流式响应（SSE），需加 --no-buffer 与 "stream": true
curl -X POST "https://llm.thunisoft.com/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer 您的应用代码" \
  --no-buffer \
  -d '{
    "model": "my-gpt-3.5",
    "messages": [
      {"role": "user", "content": "请分3点介绍深度学习。"}
    ],
    "stream": true
  }'
```

4) 函数调用示例（Tools / Function Calling）

先发送包含 `tools` 的请求，模型返回 `tool_calls` 后，在本地执行函数，再把结果以 `role=tool` 回传，最后获取最终答案。

cURLJavaScriptPython复制代码

```
import json
from openai import OpenAI

client = OpenAI(
    base_url="https://llm.thunisoft.com/v1",
    api_key="您的应用代码"
)

messages = [{"role": "user", "content": "帮我查一下北京今天的天气"}]
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "根据城市查询天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {"type": "string", "description": "城市名"}
                },
                "required": ["city"]
            }
        }
    }
]

first = client.chat.completions.create(
    model="my-tool-model",
    messages=messages,
    tools=tools,
    tool_choice="auto"
)

msg = first.choices[0].message
if not msg.tool_calls:
    print(msg.content)
    raise SystemExit

tool_call = msg.tool_calls[0]
args = json.loads(tool_call.function.arguments or "{}")

# 本地函数执行（示例）
tool_result = {"city": args.get("city"), "weather": "晴，18~26°C"}

second = client.chat.completions.create(
    model="my-tool-model",
    messages=[
        *messages,
        msg,
        {
            "role": "tool",
            "tool_call_id": tool_call.id,
            "content": json.dumps(tool_result, ensure_ascii=False),
        }
    ],
)

print(second.choices[0].message.content)
```

5) 模型列表接口（/api/v1/models）

返回模型广场中已发布模型列表（OpenAI 兼容结构），并包含上下文长度、输入价格、输出价格等元数据。

- 请求方式：`GET /api/v1/models`
- 认证头：`Authorization: Bearer 您的应用代码 API Key`（必传）
- 返回结构：`object=list`，`data[]` 中含 `id/context_length/input_price/output_price`

```
# 获取可用模型列表（仅返回已发布模型）
curl -X GET "https://llm.thunisoft.com/api/v1/models" \
  -H "Authorization: Bearer 您的应用代码"

# 别名路径调用（等价）
curl -X GET "https://llm.thunisoft.com/v1/models" \
  -H "Authorization: Bearer 您的应用代码"
```

6) 用量统计接口（/api/v1/usage）

用于查询当前应用在统计期内的输入/输出 Token、总 Token、总费用等指标，支持按日/周/月聚合。

- 请求方式：`GET /api/v1/usage`
- 认证头：`Authorization: Bearer 您的应用代码 API Key`（必传）
- 可选参数：`model_name`、`start_time`、`end_time`、`interval(day/week/month)`

```
# 用量统计查询（按日）
curl -G "https://llm.thunisoft.com/api/v1/usage" \
  -H "Authorization: Bearer 您的应用代码" \
  --data-urlencode "start_time=2026-03-01" \
  --data-urlencode "end_time=2026-03-31" \
  --data-urlencode "interval=day"

# 按模型过滤 + 按周聚合
curl -G "https://llm.thunisoft.com/api/v1/usage" \
  -H "Authorization: Bearer 您的应用代码" \
  --data-urlencode "model_name=GLM-5" \
  --data-urlencode "start_time=2026-03-01" \
  --data-urlencode "end_time=2026-03-31" \
  --data-urlencode "interval=week"
```

常见问题排查

- 401/403：检查 Authorization、应用状态、模型授权关系
- 429 `Too Many Requests`：已触发用户端（工作空间维度）模型 Token 限流，请查看上文「用户端模型 Token 限流与 429」及 应用管理 → 模型 Token 限流 中各模型剩余额度，降并发、退避重试
- 404：检查请求路径是否为 `/v1/chat/completions`
- 模型不支持图片/工具：更换为支持对应能力的模型再调用
- 响应超时：降低输入长度、减少并发、使用流式输出并加重试机制
- 如有问题请联系技术支持
